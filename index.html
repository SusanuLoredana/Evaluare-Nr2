Raport Evaluare Nr.2

Autor: Susanu • Data: 05 Dec 2025 • Durata: 50 min

I. Commit

1. Commit file settings.xml

git add settings.xml
    git commit -m "Commit file settings.xml"

2. Redenumire în configuration.xml și amend

git mv settings.xml configuration.xml
    git commit -m "Commit configuration.xml"
    git commit --amend -m "Commit file configuration.xml"

3. Push la remote

git push origin main

II. Commit

4. Adăugare remote page1.html și revenire la commit anterior

git log --oneline
    git reset --hard <HASH>
    git push --force origin main

I. Branch

5. Creare branch stage și commit hosts.xml

git checkout -b stage
    printf "<hosts>\n  <host>localhost</host>\n</hosts>\n" > hosts.xml
    git add hosts.xml
    git commit -m "Commit file hosts.xml"
    git push

6. Merge stage → main

git checkout main
    git merge stage --no-ff -m "Merge stage into main"
    git push origin main

II. Branch

7. Creare branch dev și commit env.txt

git checkout -b dev
    printf "ENV=production\nAPI_URL=https://example.com\n" > env.txt
    git add env.txt
    git commit -m "Commit file env.txt"

8. Merge dev → stage

git checkout stage
    git merge dev --no-ff -m "Merge dev into stage"
    git push origin stage

III. Branch

9. Redenumire dev → develop și push

git branch -m dev develop
    git checkout develop
    git push -u origin develop
    git push origin --delete dev
